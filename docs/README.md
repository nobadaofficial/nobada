# Nobada - AI 연애 시뮬레이션 챗 서비스

## 프로젝트 개요

**Nobada(노바다)**는 실사 기반 숏폼 영상과 실시간 TTS(Text-to-Speech)를 결합한 차세대 AI 연애 시뮬레이션 서비스입니다.

### 핵심 특징

- 🎬 **실사 영상**: 매 대화마다 캐릭터의 실사 영상 재생
- 🎙️ **실시간 음성**: AI가 생성한 대사를 실시간 TTS로 재생
- 💕 **관계도 시스템**: 대화 내용에 따른 호감도 변화와 멀티엔딩
- 🎭 **감정 표현**: 캐릭터의 감정 상태에 맞춘 영상과 음성 매칭

## 기술 스택

### Frontend
- **Web**: Next.js 15, React 19, TypeScript, Tailwind CSS
- **Mobile**: React Native, Expo 54, NativeWind
- **Video**: Video.js, HLS.js, Web Audio API

### Backend
- **API**: Next.js API Routes
- **Database**: PostgreSQL (Neon DB), Prisma ORM
- **Cache**: Redis (Upstash)
- **Storage**: AWS S3 / Cloudflare R2

### AI/TTS
- **LLM**: OpenAI GPT-4 / Anthropic Claude
- **TTS Primary**: Naver Clova Voice (한국어 최적화)
- **TTS Fallback**: ElevenLabs Flash v2.5 (초저지연)

## 핵심 차별화 요소

### 1. 초저지연 TTS 시스템
- **목표 지연시간**: < 300ms (첫 음성 재생까지)
- **구현 방법**:
  - 스트리밍 기반 청크 단위 처리
  - 지능형 프리로드 시스템
  - 다층 캐싱 전략 (Memory → Disk → CDN)

### 2. 영상-음성 완벽 동기화
- **Web Audio API** + **MediaStream API** 활용
- 실시간 동기화 모니터링 (50ms 이내 유지)
- 적응형 비트레이트 스트리밍

### 3. AI 기반 개인화
- 대화 컨텍스트 유지 (최근 20개 대화)
- 감정 상태 추적 및 반영
- 관계도 기반 스토리 분기

## 기술적 도전 과제

### 1. TTS 지연 최소화
- **문제**: 일반 TTS는 500ms 이상의 지연
- **해결**: 스트리밍 + 프리로드 + 캐싱
- **결과**: 300ms 이내 첫 음성 재생

### 2. 영상 대역폭 관리
- **문제**: 고품질 영상의 대용량 트래픽
- **해결**: HLS 적응형 스트리밍 + CDN
- **결과**: 네트워크 상황별 최적 품질 제공

### 3. AI 응답 품질
- **문제**: 캐릭터별 일관된 페르소나 유지
- **해결**: 파인튜닝 + 컨텍스트 관리
- **결과**: 몰입감 있는 대화 경험
